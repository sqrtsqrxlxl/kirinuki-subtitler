## How to use this file

1. Testing session → every bug and idea gets one line here, immediately, no
   filtering. Don't switch into "fix it" mode mid-test — just capture and keep
   testing.
2. After the session, do one triage pass: bugs → fix soon; ideas → either stay here
   (still small/standalone) or move up into a ROADMAP.md milestone if they cluster
   with something bigger.
3. Once an item is actually built, it moves to CHANGELOG.md and gets deleted from
   here.
